# Change Management Review

## Objective

Ensure systems changes are authorized, tested, and traceable.

## Risks

* Unauthorized changes
* Downtime
* Data corruption

## Control Checklist

* Change request logged
* Approval documented
* Testing evidence
* Rollback plan

## Sample Change Log

|**CHANGE ID**|**DESCRIPTION**|**APPROVED BY**|**STATUS**|
|-|-|-|-|
|CHG001|Core banking patch|IT Manager|Approved|
|CHG002|DB config change|Security Lead|Pending|
